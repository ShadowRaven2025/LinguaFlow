# Руководство по участию в проекте LinguaFlow

Спасибо за интерес к участию в развитии LinguaFlow! Мы приветствуем любой вклад в проект.

## 🤝 Как участвовать

### Способы участия

- 🐛 **Сообщения об ошибках** - помогите нам найти и исправить баги
- 💡 **Предложения функций** - поделитесь идеями для улучшения
- 📝 **Улучшение документации** - помогите сделать документацию лучше
- 💻 **Код** - внесите изменения в код проекта
- 🎨 **Дизайн** - предложите улучшения UI/UX
- 🌍 **Переводы** - помогите локализовать приложение

## 🚀 Быстрый старт

### 1. Настройка окружения

```bash
# Форк репозитория на GitHub
# Клонирование вашего форка
git clone https://github.com/your-username/linguaflow.git
cd linguaflow

# Установка зависимостей
bun install

# Настройка переменных окружения
cp .env.example .env.local
# Заполните .env.local своими данными Supabase

# Запуск в режиме разработки
bun dev
```

### 2. Создание feature branch

```bash
# Создайте новую ветку от main
git checkout -b feature/your-feature-name

# Или для исправления бага
git checkout -b fix/bug-description
```

### 3. Внесение изменений

- Следуйте существующему стилю кода
- Добавляйте тесты для новой функциональности
- Обновляйте документацию при необходимости
- Проверяйте код линтером: `bun run lint`

### 4. Коммиты

Используйте [Conventional Commits](https://www.conventionalcommits.org/):

```bash
# Примеры хороших коммитов
git commit -m "feat: добавить компонент UserProfile"
git commit -m "fix: исправить ошибку в навигации"
git commit -m "docs: обновить README"
git commit -m "style: исправить форматирование в Button"
git commit -m "refactor: переработать API клиент"
git commit -m "test: добавить тесты для SearchDialog"
```

### 5. Pull Request

```bash
# Отправьте изменения в ваш форк
git push origin feature/your-feature-name

# Создайте Pull Request на GitHub
# Заполните шаблон PR с описанием изменений
```

## 📋 Стандарты кода

### TypeScript

- Используйте строгую типизацию
- Избегайте `any`, используйте конкретные типы
- Документируйте сложные типы комментариями

```typescript
// ✅ Хорошо
interface UserProfile {
  id: string
  name: string
  email: string
  xp: number
  level: number
}

// ❌ Плохо
interface UserProfile {
  id: any
  name: any
  // ...
}
```

### React компоненты

- Используйте функциональные компоненты с хуками
- Применяйте `forwardRef` для компонентов UI
- Мемоизируйте дорогие вычисления с `useMemo`

```typescript
// ✅ Хорошо
interface ButtonProps {
  variant?: 'default' | 'destructive'
  size?: 'sm' | 'md' | 'lg'
  children: React.ReactNode
}

const Button = forwardRef<HTMLButtonElement, ButtonProps>(
  ({ variant = 'default', size = 'md', children, ...props }, ref) => {
    return (
      <button
        ref={ref}
        className={cn(buttonVariants({ variant, size }))}
        {...props}
      >
        {children}
      </button>
    )
  }
)
```

### Стилизация

- Используйте Tailwind CSS классы
- Применяйте `cn()` для условных стилей
- Следуйте дизайн-системе shadcn/ui

```typescript
// ✅ Хорошо
<div className={cn(
  "rounded-lg border p-4",
  variant === 'destructive' && "border-red-500 bg-red-50",
  className
)}>

// ❌ Плохо
<div style={{ borderRadius: '8px', padding: '16px' }}>
```

### Именование

- **Файлы**: `kebab-case.tsx`
- **Компоненты**: `PascalCase`
- **Функции**: `camelCase`
- **Константы**: `UPPER_SNAKE_CASE`
- **Типы**: `PascalCase` с суффиксом `Type` или `Props`

## 🧪 Тестирование

### Обязательные тесты

- Unit тесты для утилит и хуков
- Component тесты для UI компонентов
- Integration тесты для критичной функциональности

```typescript
// Пример теста компонента
import { render, screen, fireEvent } from '@testing-library/react'
import { Button } from '@/components/ui/button'

describe('Button', () => {
  it('renders correctly', () => {
    render(<Button>Click me</Button>)
    expect(screen.getByRole('button')).toBeInTheDocument()
  })

  it('handles click events', () => {
    const handleClick = jest.fn()
    render(<Button onClick={handleClick}>Click me</Button>)
    
    fireEvent.click(screen.getByRole('button'))
    expect(handleClick).toHaveBeenCalledTimes(1)
  })
})
```

### Запуск тестов

```bash
# Все тесты
bun test

# Тесты в watch режиме
bun test:watch

# Покрытие кода
bun test:coverage
```

- `docs/` - соответствующие разделы документации

### Стиль документации

- Используйте ясный и простой язык

## 🏗️ Архитектурные решения

### Принципы

1. **Простота** - предпочитайте простые решения сложным
2. **Переиспользование** - создавайте переиспользуемые компоненты
3. **Производительность** - оптимизируйте критичные пути
4. **Доступность** - следуйте WCAG рекомендациям
5. **Безопасность** - всегда думайте о безопасности данных

### Структура компонентов

```
components/
├── ui/                 # Базовые UI компоненты (shadcn/ui)
├── features/           # Функциональные компоненты
├── layouts/            # Layout компоненты
└── forms/              # Формы и валидация
```

### Управление состоянием

- **Локальное состояние**: `useState`, `useReducer`
- **Серверное состояние**: Supabase real-time
- **Глобальное состояние**: Zustand (при необходимости)
- **Формы**: React Hook Form + Zod

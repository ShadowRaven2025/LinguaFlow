# ИНСТРУКЦИЯ: Как увидеть изменения

## Проблема
Браузер кэширует старые стили CSS и не показывает новые изменения.

## Решение (выполните ВСЕ шаги):

### 1. Остановите сервер разработки
- Нажмите `Ctrl + C` в терминале где запущен сервер

### 2. Очистите кэш Next.js
Выполните в PowerShell:
```powershell
Remove-Item -Path ".next" -Recurse -Force
Remove-Item -Path "node_modules/.cache" -Recurse -Force -ErrorAction SilentlyContinue
```

### 3. Запустите сервер заново
```powershell
npm run dev
```

### 4. Очистите кэш браузера (ОБЯЗАТЕЛЬНО!)

#### Способ 1 (Рекомендуется):
1. Откройте DevTools: нажмите `F12`
2. Правый клик на кнопке обновления (слева от адресной строки)
3. Выберите "Очистить кэш и жесткая перезагрузка" (Empty Cache and Hard Reload)

#### Способ 2:
1. Нажмите `Ctrl + Shift + Delete`
2. Выберите "Кэшированные изображения и файлы"
3. Выберите "За все время"
4. Нажмите "Удалить данные"
5. Перезагрузите страницу: `Ctrl + Shift + R`

#### Способ 3 (Если ничего не помогло):
1. Откройте DevTools (`F12`)
2. Перейдите на вкладку "Network" (Сеть)
3. Поставьте галочку "Disable cache" (Отключить кэш)
4. Обновите страницу

### 5. Проверьте что изменения применились

Откройте любой урок и проверьте в DevTools (F12) → Elements:
- Должны быть классы `.glass`, `.glass-dark`
- Фон должен быть `#101922`
- Должен быть элемент с классом `.gradient-bg`

## Если ВСЕ ЕЩЕ не работает:

### Попробуйте режим инкогнито:
1. Нажмите `Ctrl + Shift + N` (Chrome) или `Ctrl + Shift + P` (Firefox)
2. Откройте `http://localhost:3000`
3. Если там работает - значит проблема в кэше обычного браузера

### Или используйте другой браузер:
- Если используете Chrome, попробуйте Firefox
- Если используете Firefox, попробуйте Chrome
